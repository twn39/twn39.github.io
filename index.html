Python Tkinter 布局管理

Apr 13th, 2014 | Comments

在Tkinter中有三种方式可以对组件进行布局管理，它们分别是pack,grid,place。

pack: 由简单的组件组成的应用程序可以用pack布局，而且通过设置参数可以实现随窗口缩放的组件。通常与frame搭配也可以实现复杂的布局管理，但是没有grid清晰与简洁。

grid： 对于由许多个组件组成的复杂的程序用grid布局，效率比较高。但是无法实现组件随窗口而缩放。

place： place用于对组件进行精确的布局，place由像素制定位置，应用不多。

pack

pack使用比较简单，基本在组件名称后加上pack()函数就实现打包。

语法：

PHP Password_hash 函数的使用

Apr 10th, 2014 | Comments

在上一篇文章中，我使用sha1函数混合盐值进行加密，后来发现其实根本没有必要，因为我对这种加密方式的安全性一无所知，也不知道如何进行测试，因为我并不是个密码学家，这种方式仅仅是看起来比较安全而已。然而加密的重要的一条规则就是：不要使用未经过安全测试的加密函数！

然而我也提到了在PHP5.5中引入了比较好的加密函数password_hash()，而且在PHP5.3.8版本有一个相兼容的库。现在主流的空间都基本支持PHP5.3以上，因此password_hash()可以放心使用。

在password_hash库中有4个函数，常用的主要有3个：

  • password_hash
  • password_verify
  • password_get_info

加密函数password_hash语法：

string password_hash ( string $password , integer $algo [, array $options ] )

虽然在password_hash函数的可选参数中可以设置盐值与递归层级，但是在不了解算法内容的情况下，不建议设置。函数返回一个包含60个字符的字符串。

比对函数password_verify语法：

boolean password_verify ( string $password , string $hash )

PHP通过盐值加密

Apr 5th, 2014 | Comments

最近看了一篇PHP加密的文章，获益良多，因为无论用PHP构建任何系统，都需要对敏感的数据进行加密。而且这几年，时不时地爆出某某网站泄露用户数据什么的，足见构建一个安全的系统也不是这么容易的。

在文章中有几个重要的注意事项：

  • 不要自己写加密函数
  • 不要单纯使用低端的哈希函数，如（MD5，SHA1）
  • 不要两次哈希或者组合哈希函数

通常目前流行的方式是对密码加盐(salt)，而且盐值不能过短，必须由伪随机数生成，一个好的盐值必须与哈希函数的输出的长度相当。

那么在PHP中如何采用盐值加密？首先必须生成盐值，在PHP中有一个函数为：openssl_random_pseudo_bytes。它是由openssl扩展提供的随机数生成器，因此在PHP环境中启用openssl扩展，因为基本绝大多数的空间都有这个扩展，所以不用担心无法使用。它的语法为：

string openssl_random_pseudo_bytes ( int $length [, bool &$crypto_strong ] )

它的输入为长度，单位是（位），返回一个字符串。有了盐值就可以对密码进行加密了。

将盐值存入数据库中的用户表中，校验的时候提取盐值，就可以与原密码进行比对。 如果需要进一步加强安全性，可以将盐值放到配置文件中，作为一个公有的盐值，采用：

sha1（sha1（公有盐值+密码）+私有盐值）

据我所知WordPress就是将盐值放在配置文件中的。

在PHP5.5中，openssl扩展引入了新的函数叫做openssl_pbkdf2，使用起来相对简单，但是安全性更高，不过考虑到目前基本使用PHP5.3版本，也就不做介绍了。

更新

在PHP5中引入了新的加密函数：password_hash(),虽然在官方的5.3版本中还没实现，但他推荐了一个与5.3兼容的库，用法与php5.5手册中的相同，github地址：password_compat。

Wordpress没搞头，写博还是静态的好

Apr 4th, 2014 | Comments

为何需要写博客

对于学编程的人来说，我觉得写博客还是有必要的，写博客可以记录一些自己的想法，创意以及自己学习到的经验。像我这种过目就忘的人，尤其重要。

如何选择博客工具

其实一开始我就是个WordPress的粉丝，也是因为WordPress才开始接触PHP的。像当年（也就一两年吧）我在狗爹（godaddy）斥巨资买了3个月的空间，就是为了搭建一个WordPress个人博客，那时候找各种主题，各种插件，花了几天总算搭建了一个比较满意的博客，后来也就写了几篇博客就搁在那里，再也没去动过它。

再后来我发现了一个叫Markdown的东东，我心里咒骂道：以前我怎么没发现这个东西呢。Markdown跟Wordpress就好比Tex跟Word，只要专注于内容而非板式与页面。后来就基本没碰过WordPress了，就只使用StackEdit+Dropbox了。
