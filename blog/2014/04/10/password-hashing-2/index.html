PHP Password_hash 函数的使用

Apr 10th, 2014 | Comments

在上一篇文章中，我使用sha1函数混合盐值进行加密，后来发现其实根本没有必要，因为我对这种加密方式的安全性一无所知，也不知道如何进行测试，因为我并不是个密码学家，这种方式仅仅是看起来比较安全而已。然而加密的重要的一条规则就是：不要使用未经过安全测试的加密函数！

然而我也提到了在PHP5.5中引入了比较好的加密函数password_hash()，而且在PHP5.3.8版本有一个相兼容的库。现在主流的空间都基本支持PHP5.3以上，因此password_hash()可以放心使用。

在password_hash库中有4个函数，常用的主要有3个：

  • password_hash
  • password_verify
  • password_get_info

加密函数password_hash语法：

string password_hash ( string $password , integer $algo [, array $options ] )

示例：

虽然在password_hash函数的可选参数中可以设置盐值与递归层级，但是在不了解算法内容的情况下，不建议设置。函数返回一个包含60个字符的字符串。

比对函数password_verify语法：

boolean password_verify ( string $password , string $hash )

获取加密信息函数password_get_info,语法为：

array password_get_info ( string $hash )
